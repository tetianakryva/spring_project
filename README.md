# Attendance Tracking System

# Система відстеження відвідуваності лекцій та лабораторних робіт

Ця система призначена для відстеження відвідуваності студентів на лекціях та лабораторних роботах у навчальному закладі. Система надає можливість викладачам вести облік присутності студентів, а студентам - переглядати свій журнал відвідуваності. Адміністратори можуть управляти обліковими записами користувачів та переглядати загальну статистику відвідуваності.

### **Функціональні вимоги:**

- Як викладач, я хочу мати можливість створити свій обліковий запис з іменем, прізвищем та електронною поштою.
- Як викладач, я хочу мати можливість створювати записи про заняття, включаючи назву курсу, номер групи, дату, кількість аудиторії та тип заняття (лекція чи лабораторна робота).
- Як викладач, я хочу мати можливість відзначати присутність або відсутність студентів на занятті.
- Як викладач, я хочу мати можливість вносити причину відсутності студента, а також відзначати, чи буде студент відпрацьовувати пропущене заняття.
- Як студент, я хочу мати можливість створити свій обліковий запис з іменем, прізвищем, групою та електронною поштою.
- Як студент, я хочу мати можливість переглядати свій журнал відвідуваності занять.
- Як адміністратор, я хочу мати можливість управляти обліковими записами викладачів та студентів.
- Як адміністратор, я хочу мати можливість переглядати загальну статистику відвідуваності занять.

### **System Behavior**

1. **Створення облікових записів:**
   - Викладачі та студенти можуть створювати облікові записи, вводячи необхідну інформацію (ім'я, прізвище, електронна пошта тощо).
   - Система забезпечує конфіденційність інформації про облікові записи користувачів за допомогою захищених протоколів аутентифікації та шифрування даних.

2. **Управління заняттями (для викладачів):**
   - Викладачі можуть створювати записи про заняття, вказуючи назву курсу, номер групи, дату, кількість аудиторії та тип заняття (лекція чи лабораторна робота).
   - Під час заняття викладач може відзначати присутність або відсутність студентів.
   - У разі відсутності студента викладач може вказати причину та зазначити, чи буде студент відпрацьовувати пропущене заняття.
   - Система надсилає сповіщення студентам про необхідність відпрацювання пропущеного заняття.

3. **Перегляд журналу відвідуваності (для студентів):**
   - Студенти можуть переглядати свій журнал відвідуваності занять, щоб відстежувати свої пропуски та відпрацювання.
   - Журнал відвідуваності містить дані про дату заняття, тип заняття, присутність або відсутність студента, причину відсутності (за наявності) та статус відпрацювання (за необхідності).

4. **Адміністрування системи:**
   - Адміністратори можуть управляти обліковими записами викладачів та студентів (створення, редагування, видалення).
   - Адміністратори можуть переглядати загальну статистику відвідуваності занять для моніторингу та аналізу.
   - Статистика відвідуваності включає дані про відсоток присутності студентів на заняттях, кількість пропусків, відпрацювань тощо.

5. **Інтеграція з іншими системами:**
   - Система може інтегруватися з існуючими системами управління навчальним процесом або студентськими базами даних для автоматичного імпорту інформації про студентів та курси.
   - Дані про відвідуваність можуть бути експортовані у різних форматах (CSV, Excel тощо) для подальшого аналізу та звітності.

### **API Endpoints**

| Method | Path                                  | Description                                       | Response                                           |
|--------|---------------------------------------|----------------------------------------------------|----------------------------------------------------|
| POST   | /api/teachers                         | Створення облікового запису викладача            | Підтвердження або повідомлення про помилку        |
| POST   | /api/students                         | Створення облікового запису студента             | Підтвердження або повідомлення про помилку        |
| GET    | /api/teachers/{teacher_id}            | Отримати інформацію про викладача                | Об'єкт JSON з інформацією про викладача           |
| GET    | /api/students/{student_id}            | Отримати інформацію про студента                 | Об'єкт JSON з інформацією про студента            |
| POST   | /api/classes                          | Створення запису про заняття (для викладачів)    | Підтвердження або повідомлення про помилку        |
| PUT    | /api/attendance                       | Оновлення відвідуваності на занятті               | Підтвердження або повідомлення про помилку        |
| GET    | /api/attendance/{student_id}          | Отримати журнал відвідуваності студента          | Масив об'єктів JSON з записами про відвідуваність |
| GET    | /api/statistics                       | Отримати загальну статистику відвідуваності      | Об'єкт JSON зі статистичними даними               |
| PUT    | /api/attendance/makeup                | Відзначити відпрацювання пропущеного заняття     | Підтвердження або повідомлення про помилку        |
| DELETE | /api/attendance/{attendance_id}       | Видалити запис про відвідуваність                | Підтвердження або повідомлення про помилку        |
